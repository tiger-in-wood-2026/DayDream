# 2048 手势控制游戏设计文档

## 概述
基于 OpenCV 的 2048 游戏，通过摄像头实时检测手指滑动方向来控制游戏合并操作。

## 技术栈
- Python 3.12 + opencv-python 4.13.0
- 单文件实现，无额外依赖

## 架构

### 1. 2048 游戏引擎 (`Game2048` 类)

**数据结构：**
- `board[4][4]`: 整型二维数组，0 表示空格，2 的幂表示 tile 值
- `score`: 当前分数
- `moved`: 每次操作是否有变化标记

**核心接口：**
| 方法 | 说明 |
|------|------|
| `move(direction)` | 方向 `up/down/left/right`，执行合并逻辑 |
| `add_random_tile()` | 在空位随机生成 2 或 4 |
| `is_game_over()` | 检测是否无空位且无法合并 |
| `reset()` | 重置游戏 |
| `get_board()` | 返回当前棋盘状态 |

**合并逻辑：** 按方向遍历，逐行/列压缩 + 相邻相同合并 + 再次压缩，经典 2048 算法。

### 2. 渲染模块

使用 OpenCV `cv2.rectangle()` 和 `cv2.putText()` 绘制：
- 4x4 棋盘，每个 tile 根据值显示不同颜色
- 分数显示
- 右下角摄像头小窗（缩放到 200x150）

### 3. 手势检测模块

**肤色检测：** HSV 色彩空间阈值 (H:0-30, S:30-150, V:50-255)，针对东亚肤色优化。

**手部定位：**
1. 高斯模糊去噪
2. 肤色掩膜 + 形态学开闭运算
3. 找最大轮廓作为手部
4. 计算凸包 + 凸缺陷
5. 指尖检测：凸包顶点中，曲率最大的点（通常与腕部方向相反的最高点）

**滑动检测：**
- 记录指尖最近 10 帧位置
- 计算平均运动向量
- 水平方向阈值 > 40px，垂直方向阈值 > 30px
- 方向锁定：选择位移最大的轴
- 防抖：触发后 500ms 内忽略新操作

### 4. 主循环

```
while running:
    1. cap.read() → frame
    2. 缩小小窗 → flip → 检测手势
    3. 如果有滑动操作: game.move(direction)
    4. 渲染游戏棋盘 (主画面)
    5. 将摄像头小窗贴到右下角
    6. imshow() 显示
    7. waitKey(30) — ESC退出
```

## 文件结构
- `2048_game.py` — 主程序（游戏引擎 + 手势检测 + 渲染 + 主循环）

## 操作说明
| 手势 | 游戏操作 |
|------|----------|
| 指尖左滑 | ← 向左合并 |
| 指尖右滑 | → 向右合并 |
| 指尖上滑 | ↑ 向上合并 |
| 指尖下滑 | ↓ 向下合并 |
| 按 ESC 键 | 退出程序 |

## 已知限制
- 自然肤色检测受光照条件影响，建议在均匀光照下使用
- 单一摄像头，背景尽量简洁
- 无 MediaPipe 等高精度手部关键点检测，指尖定位精度有限
